## 登录体验优化

**提问：** `[16:25]`
1. 点击登录后，登录按钮在加载中，同时页面也出现加载中提示，体验差
2. 加载完成后，没有立即跳转，而是先loading消失，延迟1秒左右才跳转

**修复：优化登录加载体验** `[16:25]`

### 问题原因

- 双重loading：登录按钮loading + 页面loading（wx.showLoading）
- 延迟跳转：登录成功后显示提示，延迟1秒才跳转

### 修改文件

1. **miniprogram/pages/login/login.js**
   - 移除 `isLoading` 状态
   - 移除登录按钮loading相关逻辑
   - 保留页面级 `wx.showLoading()`
   - 移除成功提示，登录成功直接跳转
   - 移除 `setTimeout` 延迟

2. **miniprogram/pages/login/login.wxml**
   - 移除登录按钮的loading状态显示
   - 简化按钮结构

### 修复内容

```javascript
// 修复前
this.setData({ isLoading: true })
wx.showLoading({ title: '登录中...' })
// ...
wx.hideLoading()
wx.showToast({ title: '登录成功', icon: 'success' })
setTimeout(() => {
  wx.switchTab({ url: '/pages/index/index' })
}, 1000)

// 修复后
wx.showLoading({ title: '登录中...' })
// ...
wx.hideLoading()
wx.switchTab({ url: '/pages/index/index' })
```

### 优化效果

| 项目 | 修复前 | 修复后 |
|------|--------|--------|
| loading状态 | 按钮loading + 页面loading | 仅页面loading |
| 成功提示 | 显示1秒提示 | 无提示，直接跳转 |
| 跳转时机 | 延迟1秒 | 立即跳转 |
| 用户体验 | 差，操作不流畅 | 好，流畅自然 |

```javascript

  const query = wx.createSelectorQuery()
  query.selectAll('.category-item').boundingClientRect()
  query.select('.content-scroll').boundingClientRect()
  query.exec((res) => {
    const itemRects = res[0].map(rect => ({
      top: scrollRect.top + rect.top,
      bottom: scrollRect.top + rect.bottom,
      height: rect.height
    }))
    this.setData({ itemRects })
  })
}

// 拖拽移动：根据边界计算目标索引
onDragMove(e) {
  const newY = touch.clientY - this.dragOffsetY
  let targetIndex = this.data.placeholderIndex
  
  for (let i = 0; i < itemRects.length; i++) {
    const rect = itemRects[i]

      break
    }
    if (i === itemRects.length - 1) {
      targetIndex = i
    }
  }
  
  this.setData({ placeholderIndex: targetIndex })
}

## 修复拖拽排序逻辑（空位交换法）

**提问：** `[17:46]`
1. 拖拽排序时，b被瞬间移到c的位置，然后c变成被拖拽的元素
2. 继续向下拖拽时，c只在一定范围内移动，每次超过下一个都会回到初始b的位置
3. 最终结果是b跑到列表末尾（a c d e f g b），而不是预期的位置

**修复：使用空位交换法重新实现拖拽排序** `[17:46]`

### 问题原因

原始实现中，每次发生交换时，`dragStartIndex` 和 `dragStartY` 被重置为新位置，导致：
- 被拖拽的元素身份不断变化（b→c→d...）
- 下一次计算从新位置重新开始，造成"瞬间移动"效果
- 最终被拖拽的元素跑到列表末尾

### 解决方案

采用**空位交换法**：
1. 被拖拽的元素从数组中移出，单独存储（`dragItem`）
2. 在原位置显示空位占位符（`emptyIndex`）
3. 被拖拽元素通过浮动层跟随手指（`position: fixed`）
4. 移动空位时，只更新 `emptyIndex`，不修改数组顺序
5. 松开时，将被拖拽元素插入到空位位置

### 修改文件

1. **miniprogram/pages/category/category.js**
   - 新增 `dragItem`、`emptyIndex`、`dragStartY` 状态
   - 重写 `onDragStart`：从数组移除被拖拽元素，记录起始位置
   - 重写 `onDragMove`：计算目标位置，移动空位（只更新 `emptyIndex`）
   - 重写 `onDragEnd`：将被拖拽元素插入空位位置，保存排序
   - 新增 `moveEmptySlot` 方法：更新空位索引
   - 移除旧的 `swapCategories` 方法

2. **miniprogram/pages/category/category.wxml**
   - 列表中显示空位占位符（虚线边框）
   - 列表外添加浮动拖拽层（`.drag-ghost`）
   - 浮动层使用 `position: fixed` 跟随手指

3. **miniprogram/pages/category/category.wxss**
   - 新增 `.empty-slot` 样式：透明背景、虚线边框
   - 新增 `.drag-ghost` 样式：固定定位、阴影效果、半透明

### 实现原理

```
拖拽开始 → 被拖拽元素从数组移出 → 原位置显示空位占位符
    ↓
         → 被拖拽元素通过浮动层显示
    ↓
拖拽移动 → 浮动层跟随手指移动 → 根据偏移量计算目标索引
    ↓

    ↓
         → 列表中其他元素位置不变
    ↓
拖拽结束 → 将被拖拽元素插入到 emptyIndex 位置
    ↓
         → 保存排序到云数据库
```

### 关键代码

  const categories = [...this.data.currentCategories]
  const dragItem = categories.splice(index, 1)[0]

  this.setData({
    dragItem: dragItem,
    emptyIndex: index,
    currentCategories: categories,
    dragStartY: touch.clientY
  })
}

// 拖拽移动：只更新空位索引
moveEmptySlot(fromIndex, toIndex) {
  this.setData({ emptyIndex: toIndex })
}

// 拖拽结束：插入被拖拽元素
onDragEnd(e) {

  newCategories.splice(emptyIndex, 0, dragItem)
  this.setData({ currentCategories: newCategories })
}
```

## 修复拖拽排序连续跳动问题

**提问：** `[17:58]`
1. 拖拽时一旦超过某个阈值，就会自动连续交换，直到边界
2. 手指只超过了一个元素，但直接顺延排到第一个/最后一个

**修复：改为基于相对偏移计算目标位置** `[17:58]`

### 问题原因

上一次修复中，`offsetY` 始终基于**原始起始位置**计算，导致 `moveCount` 持续累积：

```
初始：dragIndex=1, dragStartY=100, itemHeight=60, threshold=0.8
effectiveHeight = 60 * 0.8 = 48

手指到 148：offsetY=48, moveCount=1, targetIndex=1+1=2 ✓
手指到 196：offsetY=96, moveCount=2, targetIndex=2+2=4 ✗（跳过了位置3！）
手指到 244：offsetY=144, moveCount=3, targetIndex=4+3=7→4 ✗（卡在最后）
```

### 解决方案

改为**基于相对偏移**计算目标位置：每次只计算手指相对于**上次位置**的偏移，而不是相对于**原始位置**的偏移。

```javascript
// 错误：基于原始位置
const offsetY = touch.clientY - this.dragStartY  // 持续累积
const moveCount = Math.round(offsetY / (itemHeight * threshold))

// 正确：基于上次位置
const lastTouchY = this.lastTouchY || this.dragStartY
const relativeOffset = touch.clientY - lastTouchY  // 每次重置
const moveCount = Math.round(relativeOffset / (itemHeight * threshold))
this.lastTouchY = touch.clientY  // 更新上次位置
```

### 修改文件

1. **miniprogram/pages/category/category.js**
   - `onDragStart`：新增 `this.lastTouchY = touch.clientY`
   - `onDragMove`：改用 `relativeOffset` 计算 `moveCount`，更新 `lastTouchY`
   - `onDragEnd`：清理 `this.lastTouchY = null`
   - 视觉跟随仍使用 `totalOffset = touch.clientY - this.dragStartY`

### 核心修复对比

| 修复点 | 修复前 | 修复后 |
|--------|--------|--------|
| 偏移量计算 | `offsetY = touch.clientY - dragStartY`（持续累积） | `relativeOffset = touch.clientY - lastTouchY`（每次重置） |
| 目标索引 | `targetIndex = currentIndex + moveCount`（moveCount 累积） | `targetIndex = currentIndex + moveCount`（moveCount 每次重新计算） |
| 视觉跟随 | `dragOffset = offsetY` | `dragOffset = totalOffset`（保持视觉正确） |

### 行为对比

**修复前（基于原始位置）：**
```
手指到 148：offsetY=48, moveCount=1, targetIndex=1+1=2 ✓
手指到 196：offsetY=96, moveCount=2, targetIndex=2+2=4 ✗（跳过位置3）
手指到 244：offsetY=144, moveCount=3, targetIndex=4+3=7→4 ✗（卡在最后）
```

**修复后（基于上次位置）：**
```
手指到 148：relativeOffset=48, moveCount=1, targetIndex=1+1=2 ✓
手指到 196：relativeOffset=48, moveCount=1, targetIndex=2+1=3 ✓
手指到 244：relativeOffset=48, moveCount=1, targetIndex=3+1=4 ✓
```

### 关键代码

```javascript
onDragMove(e) {
  const touch = e.touches[0]
  const currentIndex = this.data.dragIndex
  const itemHeight = this.data.itemHeight

  // 计算手指相对于上次位置的偏移
  const lastTouchY = this.lastTouchY || this.dragStartY
  const relativeOffset = touch.clientY - lastTouchY

  // 根据相对偏移计算目标索引（每次只移动一个位置）
  const threshold = 0.8
  const moveCount = Math.round(relativeOffset / (itemHeight * threshold))
  const targetIndex = Math.max(0, Math.min(length - 1, currentIndex + moveCount))

  // 更新拖拽偏移量（基于原始位置，用于视觉跟随）
  const totalOffset = touch.clientY - this.dragStartY
  this.setData({ dragOffset: totalOffset })

  // 更新上次触摸位置
  this.lastTouchY = touch.clientY

  // 如果位置变化，交换分类
  if (targetIndex !== currentIndex) {
    this.swapCategories(currentIndex, targetIndex)
    this.setData({ dragIndex: targetIndex })
  }
}
```

## 分类管理拖拽排序功能彻底重写

**提问：** `[23:30]`
1. 用户反馈分类管理的拖拽排序功能有问题
2. 拖拽元素无法跟随手指移动，超过其他元素时会弹跳回原位
3. 视觉偏移量计算错误，导致元素位置跳动

**修复：彻底重写拖拽排序逻辑** `[23:41]`

### 问题分析

旧方案的根本缺陷：
- 每次交换位置时都调用 `swapCategories()` + `setData({ dragIndex: targetIndex })`
- `dragIndex` 改变导致 DOM 重新渲染，被拖拽元素跳动到新位置
- 视觉偏移量基于原始位置计算，与实际位置脱节

### 新方案核心思路

**拖拽过程中不修改数组，只计算每个元素的视觉偏移量；拖拽结束后才真正重排数组。**

| 对比项 | 旧方案 | 新方案 |
|--------|--------|--------|
| 拖拽中 | 不断 swap 数组 + 改变 dragIndex | 不修改数组，只计算视觉偏移 |
| dragIndex | 每次交换都改变 | 始终不变（锁定原始位置） |
| 视觉效果 | DOM 重渲染导致元素跳动 | 每个元素独立偏移，平滑跟随手指 |
| 拖拽结束 | - | 才真正 swap 数组并保存排序 |

### 修改文件

1. **miniprogram/pages/category/category.js**
   - 新增 `itemOffsets` 数组：存储每个元素的视觉偏移量
   - `onDragStart`：初始化 `itemOffsets` 数组
   - `onDragMove`：不修改数组，只计算每个元素的偏移量
     - 被拖拽元素：`offset = clampedOffset`（跟随手指）
     - 被越过的元素：上移或下移一个元素高度
   - `onDragEnd`：根据最终偏移量计算目标位置，真正 swap 数组

2. **miniprogram/pages/category/category.wxml**
   - 样式绑定改为 `itemOffsets[index]`，每个元素独立偏移
   - `z-index` 动态设置，被拖拽元素置顶

### 新方案实现原理

```
手指向下拖拽第2项到第4项的位置：

原始顺序：  [A] [B] [C] [D] [E]    dragIndex = 1
                    ↑ 手指位置

拖拽中（数组不变，只做视觉偏移）：
  A: offset = 0       （不动）
  B: offset = +60px   （跟随手指向下）
  C: offset = -40px   （被越过，上移一格）
  D: offset = -40px   （被越过，上移一格）
  E: offset = 0       （不动）

拖拽结束：真正 swap 数组 → [A] [C] [D] [B] [E]
```

### 测试结果

拖拽功能完美实现：
- 被拖拽元素始终跟随手指移动
- 被越过的元素平滑上移/下移
- 拖拽结束后正确保存排序
- 无跳动、无弹跳问题
