**修复：修复年度总结数据加载** `[16:08]`

### 问题原因

数据库中的 `date` 字段存储的是**字符串格式**（如 "2026-01-15"），而不是时间戳。使用 `_.gte(startDate).and(_.lte(endDate))` 查询时间戳范围无法匹配字符串格式的日期。

### 解决方案

1. 获取所有账单数据
2. 在前端过滤当年数据（使用 `bill.date.startsWith(String(year))`）
3. 确保金额默认显示 0.00

### 修改文件

1. **miniprogram/pages/yearly-summary/yearly-summary.js**
   - 移除日期范围查询
   - 获取所有账单后在前端过滤当年数据
   - 使用 `bill.date.startsWith(String(year))` 判断年份

2. **miniprogram/pages/yearly-summary/yearly-summary.wxml**
   - 确保金额显示 0.00 而不是空
   - 使用三元运算符处理 null 值

3. **miniprogram/pages/mine/mine.js**
   - 同步修复年度总结数据加载逻辑

### 数据过滤逻辑

```javascript
// 过滤当年数据
allBills.forEach(bill => {
  if (bill.date && bill.date.startsWith(String(year))) {
    totalCount++
    if (bill.type === 'income') {
      totalIncome += (bill.amount || 0)
    }
    if (bill.type === 'expense') {
      totalExpense += (bill.amount || 0)
    }
  }
})
```

```javascript
formatAmount(amount) {
```
