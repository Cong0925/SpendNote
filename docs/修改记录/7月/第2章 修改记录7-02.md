**修复：记账逻辑修改（支出/收入分离）** `[09:32]`
- 问题：原逻辑允许收入选择负债账户，导致负债账户余额可正可负，逻辑混乱
- 修复方案：
  1. 修改 `miniprogram/pages/add/add.js` 中的 `loadAccountList` 方法
     - 支出时：显示所有账户（普通+负债）
     - 收入时：只显示普通账户（过滤掉 `isDebt: true` 的账户）
  2. 修改 `miniprogram/pages/add/add.js` 中的 `switchType` 方法
     - 切换收入/支出时，清空已选择的账户并重新加载账户列表
  3. 修改 `cloudfunctions/billFunctions/index.js` 中的 `updateAccountBalance` 方法
     - 负债账户：支出时负债增多（余额增加），收入已在前端过滤不允许选择
     - 普通账户：收入增加余额，支出减少余额（可为负数，允许透支）
- 逻辑说明：
  - 支出：普通账户余额可为负数（透支），负债账户负债增多
  - 收入：只能选择普通账户，负债账户的还款通过转账界面进行
- 改动文件：
  1. miniprogram/pages/add/add.js
  2. cloudfunctions/billFunctions/index.js

**修复：记账逻辑与账户显示修复** `[09:24]`
- 问题1：记账逻辑错误
  - 当前行为：选择收入 + 负债账户 → 负债金额增加（错误）
  - 正确行为：选择收入 + 负债账户 → 负债金额减少（收入抵消负债）
  - 原因：负债账户的余额是负数，收入应该让它变得更接近零
- 问题2：账户选择弹窗显示错误
  - 当前行为：负债账户余额显示为正数（如 5000 元）
  - 正确行为：负债账户余额应显示为负数（如 -5000 元）
- 修复方案：
  1. 修改 `cloudfunctions/billFunctions/index.js` 中的 `updateAccountBalance` 函数
     - 新增获取账户信息逻辑，判断 `isDebt` 字段
     - 负债账户：收入时减少负债（余额减小），支出时增加负债（余额增大）
     - 普通账户：收入时增加余额，支出时减少余额
  2. 修改 `miniprogram/components/account-picker/index.wxml`
     - 负债账户余额显示为负数：`{{item.isDebt ? '-' : ''}}{{item.balance}}`
  3. 修改 `miniprogram/pages/account/account.wxml`
     - 账户列表中负债账户显示负数：`￥-{{item.balance}}`
  4. 修改 `miniprogram/pages/accountInfo/accountInfo.wxml`
     - 账户详情中负债账户显示负数：`{{account.isDebt ? '￥-' : '￥'}}`
- 改动文件：
  1. cloudfunctions/billFunctions/index.js
