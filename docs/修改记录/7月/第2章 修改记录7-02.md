**修复：账单删除功能不生效（action名称错误+响应未校验）** `[14:59]`
- 问题：删除账单后显示"删除成功"，但页面数据仍在，接口仍返回该账单
- 根因分析：
  1. **index.js 和 detail.js 的 action 名称错误**：调用 `action: 'deleteBill'`，但云函数定义的是 `action: 'delete'`，导致走 default 分支返回"未知操作类型"

- 修复方案：
  1. 修正 index.js 的 action 名称为 `'delete'`，参数改为 `data: { id: billId }`
  2. 修正 detail.js 的 action 名称为 `'delete'`（上一轮已修复）
  3. 两处都添加 `res.result.success` 检查，失败时显示错误信息
  4. 云函数 `deleteBill` 添加 `result.stats.removed === 0` 检查，未删除时返回失败

**修复：资金链路完整修复（借出/借入/还款联动账户余额）** `[15:20]`
- 问题：借出/借入记录创建时没有联动更新账户余额，导致资金链路不完整
- 根本原因：
  1. `addLoan.js` 创建借出/借入记录时，只调用 `loanFunctions.add`，没有同时更新账户余额
  2. `loanFunctions.updatePaidAmount` 还款时没有联动更新账户余额
  3. `accountFunctions.getAccountSummary` 只计算账户余额，没有包含借出应收和借入应还
- 修复方案：
  1. **借出/借入时联动更新账户余额**（`addLoan.js`）
     - 创建借出记录时，同时减少对应账户余额（负数）
     - 创建借入记录时，同时增加对应账户余额（正数）
     - 添加账户选择必填校验
  2. **还款时联动更新账户余额**（`loanFunctions/index.js`）
     - 借出还款（收到还款）：增加账户余额（正数）
     - 借入还款（还给别人）：减少账户余额（负数）
  3. **删除借款记录时联动更新账户余额**（`loanFunctions/index.js`）
     - 删除借出记录：增加账户余额（恢复未还部分）
     - 删除借入记录：减少账户余额（扣除未还部分）
  4. **总资产计算包含借出应收和借入应还**（`accountFunctions/index.js`）
     - 总资产 = 账户余额 + 借出应收
     - 总负债 = 负债账户余额 + 借入应还
     - 净资产 = 总资产 - 总负债
- 资金链路完整性验证：
  - 借出：账户余额减少 → 借出应收增加 → 总资产不变
  - 借入：账户余额增加 → 借入应还增加 → 净资产不变
  - 还款（借出）：账户余额增加 → 借出应收减少 → 总资产不变
  - 还款（借入）：账户余额减少 → 借入应还减少 → 净资产不变
  - 删除借款：反向恢复账户余额 → 借出/借入金额减少 → 总资产/净资产恢复
