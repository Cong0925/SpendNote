# 邮件发送配置说明

## 概述

问题反馈功能支持将用户反馈自动发送到指定邮箱。使用 **163邮箱 SMTP** 作为邮件发送服务。

## 配置步骤

### 1. 准备163邮箱账号

1. 登录 [163邮箱](https://mail.163.com/)
2. 进入「设置」→「POP3/SMTP/IMAP」
3. 开启「POP3/SMTP 服务」
4. 生成**客户端授权密码**（不是邮箱登录密码）

> ⚠️ 重要：必须使用客户端授权密码，不是邮箱登录密码

### 2. 在云开发控制台配置环境变量

1. 打开微信开发者工具
2. 点击「云开发」按钮
3. 进入「设置」→「环境变量」
4. 添加以下环境变量：

| 变量名 | 值 | 说明 |
|-------|---|------|
| `SMTP_USER` | `18515155921@163.com` | 163邮箱账号 |
| `SMTP_PASS` | `FBbqD6UEtvwPD3T2` | 客户端授权密码 |

### 3. 部署云函数

1. 右键点击 `cloudfunctions/feedbackFunctions`
2. 选择「上传并部署：云端安装依赖」
3. 等待部署完成

## 工作原理

### 定时触发器

云函数配置了定时触发器，每天早上9点自动执行：

```json
{
  "triggers": [
    {
      "name": "emailTrigger",
      "type": "timer",
      "config": "0 0 9 * * * *"
    }
  ]
}
```

### 发送流程

1. 查询所有 `emailSent: false` 的反馈（最多100条）
2. 按用户分组
3. 获取用户信息
4. 下载图片并转换为base64
5. 每封邮件最多50条反馈，超过分批发送
6. 生成 HTML 邮件内容
7. 调用163邮箱 SMTP 发送邮件
8. 标记 `emailSent: true`

### 邮件配置

```javascript
// SMTP 配置
{
  host: 'smtp.163.com',
  port: 465,
  secure: true,  // 使用SSL
  auth: {
    user: process.env.SMTP_USER,
    pass: process.env.SMTP_PASS
  }
}
```

### 邮件内容

邮件包含：
- 用户信息（昵称、头像）
- 反馈类型、内容、截图（base64嵌入）
- 反馈编号、提交时间
- 联系方式（如有）

### 图片处理

**问题：** 云存储的图片URL无法被外部访问

**解决方案：** 将图片下载并转换为base64格式，直接嵌入邮件

```javascript
// 1. 从云存储下载图片
const fileRes = await cloud.downloadFile({ fileID: imageUrl })

// 2. 转换为base64
const base64 = fileRes.fileContent.toString('base64')

// 3. 嵌入邮件
<img src="data:image/png;base64,${base64}">
```

**优点：**
- 图片直接显示，无需外部访问
- 邮件客户端兼容性好
- 保护图片隐私

**缺点：**
- 邮件体积增大
- 每封邮件大小限制 30MB

### 分批发送

每封邮件最多包含50条反馈：

```javascript
const maxFeedbacksPerEmail = 50

for (let i = 0; i < feedbacks.length; i += maxFeedbacksPerEmail) {
  const batch = feedbacks.slice(i, i + maxFeedbacksPerEmail)
  await sendEmail(batch)
}
```

**示例：**
- 45条反馈 → 发送1封邮件（45条）
- 80条反馈 → 发送2封邮件（50条 + 30条）

## 安全说明

- SMTP 授权密码存储在云函数环境变量中，不会暴露给前端
- 邮件发送在云函数中执行，用户无法直接调用
- 只有未发送的反馈会被处理

## 常见问题

### Q: 邮件没有发送？

A: 检查以下几点：
1. 环境变量 `SMTP_USER` 和 `SMTP_PASS` 是否配置正确
2. 163邮箱是否已开启 POP3/SMTP 服务
3. 客户端授权密码是否正确（不是登录密码）
4. 云函数是否部署成功
5. 定时触发器是否正常工作

### Q: 如何手动触发邮件发送？

A: 可以在云开发控制台手动调用云函数：
```javascript
wx.cloud.callFunction({
  name: 'feedbackFunctions',
  data: { action: 'sendPendingEmails' }
})
```

### Q: 如何查看发送记录？

A: 163邮箱没有专门的发送记录页面，但可以检查：
1. 收件邮箱是否收到邮件
2. 云函数日志是否有发送成功记录

### Q: 发送失败怎么办？

A: 常见原因及解决方法：
1. **授权密码错误** → 重新生成客户端授权密码
2. **发送频率过高** → 等待一段时间后重试
3. **邮件内容过大** → 减少图片数量或压缩图片

## 注意事项

1. 163邮箱需要保持 POP3/SMTP 服务开启
2. 客户端授权密码可能过期，需要定期检查
3. 每日发送限制约50封，足够大多数小程序使用
4. 邮件会出现在收件箱，注意定期清理
