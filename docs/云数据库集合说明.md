# 云数据库集合说明

## 需要创建的集合

### 1. users 集合

**用途：** 存储用户信息

**创建步骤：**
1. 打开微信开发者工具
2. 点击左上角「云开发」按钮
3. 进入「数据库」标签
4. 点击「+」按钮创建新集合
5. 输入集合名称：`users`
6. 点击确定

**字段说明：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| _id | String | 自动生成的唯一标识 |
| _openid | String | 用户openid（自动关联） |
| avatarUrl | String | 用户头像URL |
| nickName | String | 用户昵称 |
| createTime | Date | 注册时间 |
| lastLoginTime | Date | 最后登录时间 |

**权限设置：**
- 集合权限：仅创建者可读写
- 或根据实际需求调整

## 注意事项

1. 首次使用云数据库需要在云开发控制台开通
2. 确保云环境ID与 app.js 中的 env 配置一致
3. 建议为常用查询字段创建索引（如 _openid）

### 2. feedbacks 集合

**用途：** 存储用户反馈信息

**创建步骤：**
1. 打开微信开发者工具
2. 点击左上角「云开发」按钮
3. 进入「数据库」标签
4. 点击「+」按钮创建新集合
5. 输入集合名称：`feedbacks`
6. 点击确定

**字段说明：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| _id | String | 自动生成的唯一标识 |
| _openid | String | 用户openid（自动关联） |
| type | String | 反馈类型：bug/feature/other |
| content | String | 反馈内容 |
| images | Array | 截图URL数组 |
| contact | String | 联系方式（可选） |
| status | String | 状态：pending/processing/resolved |
| emailSent | Boolean | 邮件是否已发送 |
| emailSendTime | Date | 邮件发送时间 |
| createTime | Date | 创建时间 |
| updateTime | Date | 更新时间 |

**权限设置：**
- 集合权限：仅创建者可读写

**索引建议：**
- `_openid`：用于查询用户反馈
- `emailSent`：用于定时任务查询未发送邮件
- `createTime`：用于排序
